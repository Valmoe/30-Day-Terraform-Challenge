# What is Infrastructure as Code and Why It's Transforming DevOps

## Introduction
Modern infrastructure is no longer managed manually through dashboards and SSH sessions. Infrastructure as Code (IaC) allows engineers to define, provision, and manage infrastructure using code.

## What is Infrastructure as Code (IaC)?
Infrastructure as Code is the practice of managing and provisioning infrastructure through machine-readable configuration files instead of manual processes.

Instead of:
- Clicking in AWS Console
- Manually configuring servers

You:
- Write code
- Version it
- Deploy it consistently

## The Problem IaC Solves
Traditional infrastructure management is:
- Error-prone
- Inconsistent
- Hard to scale
- Difficult to reproduce

IaC solves this by:
- Enabling automation
- Ensuring consistency
- Supporting version control
- Improving collaboration

## Declarative vs Imperative Approaches

### Imperative Approach
You define *how* to achieve a result.

Example:
- Create server
- Install software
- Configure settings step-by-step

### Declarative Approach
You define *what* the final state should be.

Example:
- "I want 2 EC2 instances running"

Terraform handles the *how*.

## Why Terraform?

Terraform stands out because:
- Cloud-agnostic (AWS, Azure, GCP)
- Uses declarative syntax
- Maintains state for tracking infrastructure
- Supports reusable modules
- Strong ecosystem and community
